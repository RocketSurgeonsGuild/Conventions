# Data Model: Conventions Documentation Site Launch

**Feature**: `specs/001-docs-site-launch/spec.md`
**Date**: 2026-06-27

This is a documentation site feature. There is no application database. The "data model" describes the shape of content files and configuration objects that the docs system reads and writes.

---

## Content Page (Starlight frontmatter schema)

Every Markdown/MDX file under `docs/src/content/docs/` must conform to the Starlight docs schema extended with project-specific fields.

```yaml
---
# Required
title: string                  # Page title (shown in sidebar and <title>)

# Optional — Starlight built-ins
description: string            # Used in <meta> and llms.txt
draft: boolean                 # Default false; if true: excluded from prod build
template: 'doc' | 'splash'    # Default 'doc'; 'splash' for hero landing pages
hero:                          # Only for template: splash
  title: string
  tagline: string
  actions: [{ text, link, icon, variant }]
  banner:
    content: string

# Project-defined
tags: string[]                 # Matches keys in docs/tags.yml
---
```

**Validation rules**:

- `title` is mandatory — Starlight requires it; `add-api-frontmatter.mjs` injects it for generated pages.
- `draft: true` pages must not appear in sidebar navigation or search results in production builds.
- `tags` values must match a key defined in `docs/tags.yml`.

## API Reference Page (xmldocmd output)

Each generated page under `docs/src/content/docs/api/<package>/` is a Markdown file produced by `xmldocmd` then post-processed by `add-api-frontmatter.mjs`.

**Before post-processing** (xmldocmd output):

```markdown
# TypeName

Summary from XML docs.

## Methods

### MethodName(param1, param2)
```

**After post-processing** (`add-api-frontmatter.mjs`):

```markdown
---
title: TypeName
---

# TypeName

Summary from XML docs.
...
```

**Validation rules**:

- Every generated file must have frontmatter with `title` before the Astro build runs.
- Files that already have frontmatter (`content.startsWith('---')`) are skipped by the script.

## CI Workflow Shape (`.github/workflows/deploy-docs.yml`)

```yaml
name: Deploy Docs
on:
  push:
    branches: [main]
    paths: ['docs/**', 'src/**']
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - checkout
      - setup .NET (10.0.x)
      - setup Node.js 22 (cache: npm, cache-dependency-path: package-lock.json)
      - dotnet build -c Release (to produce DLLs + XML docs for all packages)
      - Install xmldocmd: dotnet tool install -g xmldocmd --version 2.9.0
      - Generate API reference: bash docs/scripts/generate-api.sh (new script)
      - Install docs deps: npm ci (workspace root)
      - Build docs: npm run build --workspace docs (with GH_API_TOKEN secret)
      - Upload Pages artifact (path: docs/dist)
  deploy:
    needs: build
    environment: github-pages
    steps:
      - actions/deploy-pages@v4
```

**Note on base path**: `astro.config.mjs` reads `process.env.GITHUB_ACTIONS` and sets `base = '/Conventions'` in CI.
