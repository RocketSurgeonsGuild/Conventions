Convention Contexts

The convention context is the the result of collecting all the known conventions for a given set of assembly. How those assemblies are collected can diff depending on on the convention context is created. Generally the convention context is created using the ConventionContextBuilder however you can implement your own IConventionContext if you wish.

Builder

A convention context is created from a ConventionContextBuilder. This class is used during the bootstrapping phase of your application.

You can add conventions manually, you can add them via attribute scanning, you can disable attribute scanning if you wish as well.

Note

The assemblies used during scanning can be added by using an AppDomain, DependencyContext, or List of assemblies.

Creating the context

A context can be created from a ConventionContextBuilder by using ConventionContext.From.

Using the context

Once the context is created you can use the context to find out all sorts of information.

Useful properties:

  • AssemblyProvider - The assembly provider can be used to get a list of assemblies
  • Logger - This is a diagnostic logger that can be used for logging details. If a logger is provided to the builder it will be used here.
  • Properties - Contains all the properties provided to the builder. This implements IServiceProvider and can be used with `ActivatorExtensions

Useful methods / extension methods:

  • Get<T>([string name]) - Get a given type from the Properties dictionary.
  • GetHostType() - Get's the given host type, as defined in the builder.
  • IsUnitTestHost() - Tests if the builder was setup for unit testing
    • This is handy to ensure different behavior during unit tests.
